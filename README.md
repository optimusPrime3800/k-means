# K-means на датасете Iris



## Общее описание

Датасет: **Ирисы Фишера (Iris)**.

Решается одна задача, состоящая из двух частей.


## Часть 1. Определение оптимального числа кластеров

**Цель:** найти оптимальное количество кластеров `k` с помощью готовых библиотек.

**Инструменты:** `sklearn` (методы локтя, силуэта и т.д.).

**Результат:** число `k`, которое выдает программа (не "я так вижу", не ручной подбор).

---

## Часть 2. Реализация K-means вручную

**Цель:** написать собственный алгоритм K-means для оптимального `k` из Части 1, **без использования библиотек, реализующих K-means**.

**Разрешённые библиотеки:** любые, кроме тех, которые содержат готовый K-means (можно `matplotlib`, `numpy`, `pandas`, `imageio` и т.д. — для отрисовки, подсчётов, GIF).

### Требования к визуализации

- На каждом шаге алгоритма выводятся **рисунки (2D-проекции)**.
- На рисунке:
  - Точки из разных кластеров — **разными цветами**.
  - Центроиды — явно выделены (например, крестиками или чёрными точками).
- Визуализируются:
  - **Сдвиг центроидов**
  - **Смена точек своих кластеров**
- **Количество рисунков = количество итераций** (на каждый шаг).
- Формат: можно отдельными картинками (перезаписывать или показывать последовательно), можно **GIF** (предпочтительно).

### Дополнительное пояснение (важно)

> Оптимальное количество кластеров определяется **строго по результатам Части 1** (методы локтя/силуэта и т.д.). Никакого "я так вижу" или ручного подбора.

---

- `k` — число из Части 1
